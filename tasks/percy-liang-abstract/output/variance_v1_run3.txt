Since compute grows much faster than web text available for language model pre-training, we ask how synthetic data augmentation can improve data efficiency by achieving lower i.i.d. validation loss on the original web distribution—not merely by transforming data into higher-quality or task-closer formats. Typically, synthetic data approaches focus on format or quality transformations, leaving underexplored whether synthetic data can better model the underlying pre-training distribution itself. We first show that mixing real web text with rephrases monotonically improves i.i.d. validation loss as the number of rephrases per document increases, with best estimates predicting that simple rephrasing requires $1.48\times$ less real data to match the baseline loss of 3.55, operating with 300M parameter models and 200M real tokens under unlimited compute. We then introduce the perspective that synthetic generations can form a single long megadocument rather than being shuffled as independent short documents, and propose two algorithms—stitched rephrasing and latent thoughts—that instantiate this view. Concretely, stitched rephrasing and latent thoughts achieve $1.64\times$ and $1.80\times$ data efficiency at 32 generations, improving upon simple rephrasing's $1.48\times$, while showing less signs of plateauing under generation count and yielding more pronounced gains on long-context validation sets. We further find that all three synthetic data recipes compose with ensembling, an already data-efficient strategy, whereas self-distillation does not compose with ensembling, demonstrating a qualitatively distinct benefit of the megadoc-style algorithms. Finally, our validation loss improvements generalize to downstream benchmarks, with latent thoughts improving average accuracy by 9% over the real data only baseline on PIQA, SciQ, and ARC Easy, showing that scaling synthetic megadocuments is a broadly applicable recipe for data-efficient pre-training in a compute-rich regime.
